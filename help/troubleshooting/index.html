Trouble shooting advice

The eCraft2Larn Unified User Interface (the UUI) works best in Chrome. If you encounter problems with other browsers please report it and switch to Chrome (or Chromium on Raspberry Pis).

If you are reading this on a Raspberry Pi that was configured by eCraft2Larn project members then report any problems to them since all the following should have been done. Click on one of the following links if you are having problems with:

  • Snap4Ardunio
  • AI Snap! Programming blocks
  • More to come

Snap4Ardunio trouble shooting

If you get a security warning about not trusting the certificate for snap4arduino.rocks, then you should be able to click on Advanced Settings and click to visit the site anyway. You should only need to do this once since the browser should remember it. The Snap4Ardunio team is aware of the problem and is fixing it.

If Snap4Ardunio launches but cannot connect to the Arduino, then first ensure that the Snap4Ardunio browser extension is installed. If the extension is installed and Snap4Ardunio still cannot connect to the Arduino on a USB port then sometimes disconnecting and then reconnecting the USB cable resets the connection. Remember it is always a good idea when finished running Snap4Ardunio to disconnect from the Arduino.
